## 강의 요약 (버전: 26.02.09_01)

### 1. 오버로딩
- 동일한 함수명을 가지며, 다른 매개변수의 갯수, 타입으로 구분될 수 있는 함수들.
- 단, 리턴 타입은 함수 구분에 해당되지 않는다.
- 컴파일 시점에서 호출되는 함수를 알 수 있다. (컴파일 시간 다형성)
- 흔히 이전에 사용되었던 생성자 중복이 사실은 생성자 오버로딩이다.

### Ex. 다형성?
- 동일한 형태에서 다양한 성격이 존재한다는 OOP의 성격, 컨셉이다.
- 다형성을 통해서 생산성, 가독성, 유지보수성이 향상될 수 있다.

### 2. 오버라이딩
- 상속관계 에서. (해당 강의 시점에선 상속에 대해서 깊이있게 나가지 않아, Object 클래스만 가지고 구현하였음.) 완전이 일치한 함수들.
- 오버라이딩을 하기 위해선 부보의 멤버 함수에는 virtual 키워드를, 자식의 멤버 함수에는 override 키워드를 소지하여야 한다.
- 오버라이딩은 오버로딩과 다르게 함수가 일치 하여야한다. (함수명, 매개변수의 타입, 순서, 갯수, 리턴타입(추후에 추가 설명))
- 런타임 시점(실행시에) 호출될 함수를 알 수 있다. (런타입 시간 다형성)

### 3. 디폴트 매개 변수
- 함수의 매개변수에 기본 값을 주는 방법으로, 객체같은 런타임에 생성하는 변수를 제외하고 디폴트 매개 변수를 할 수 있다.
- 디폴트 매개 변수는 항상 매개 변수(필수 매개 변수)뒤에 위치해야 된다.
